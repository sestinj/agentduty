# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AgentDuty, please report it responsibly.

**Email**: [security@continue.dev](mailto:security@continue.dev)

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours of your report
- **Initial assessment**: Within 5 business days
- **Resolution target**: Depends on severity, but we aim to patch critical issues as quickly as possible

## Scope

The following are in scope for security reports:

- The AgentDuty CLI (`cli/`)
- The AgentDuty web application (`web/`) and its API endpoints
- Authentication and session management
- Slack integration security (token handling, webhook verification)
- Data exposure or unauthorized access

The following are out of scope:

- Denial of service attacks
- Social engineering
- Issues in third-party dependencies (report these upstream, but let us know)

## Disclosure

We will coordinate with you on public disclosure timing. We ask that you give us reasonable time to address the issue before any public disclosure.

Thank you for helping keep AgentDuty and its users safe.
